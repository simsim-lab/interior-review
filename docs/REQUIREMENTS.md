# 요구사항서 — interior-review

이 웹의 **기본 동작 요구사항**을 정리한다. 사람이 읽고 동작을 확인하는 기준이자,
유닛/E2E 테스트의 검증 근거로 쓴다.

## 개요

인테리어 업체 미팅 준비용 웹앱. 공간별 **현재상태 / 요구사항**을 정리해 업체에게 공개하고,
**체크리스트**(업체 평가)는 관리자에게만 보인다.

- 스택: Next.js 16 · React 19 · TypeScript · Tailwind · Supabase · Vercel
- 페이지: `/current-state`(공개) · `/requirements`(공개) · `/checklist`(관리자) · `/login`

## 런타임 모드

| 모드 | 조건 | 동작 |
|------|------|------|
| Supabase | `NEXT_PUBLIC_SUPABASE_URL` + `ANON_KEY` 설정 | 실제 DB 읽기/쓰기, Auth 로그인 |
| seed | 위 env 부재(또는 쿼리 에러) | `lib/seed.ts` 데모 데이터, 쓰기는 로컬 반영만(저장 안 됨) |

- seed 모드 관리자 접근: `NODE_ENV≠production` **그리고** `NEXT_PUBLIC_ADMIN_DEV_BYPASS=1` 일 때만.
- 관리자 판정은 `ADMIN_EMAILS` allowlist 이메일만(비면 무조건 비관리자 — fail-closed).

## 요구사항

### 라우팅
- **R1** 루트 `/` 접속 시 `/current-state` 로 이동한다.

### 공개 페이지 (비로그인)
- **R2** `/current-state` 는 제목 "현재상태" 와 공간별 데이터를 렌더한다.
- **R3** `/requirements` 는 제목 "요구사항" 과 공간별 데이터를 렌더한다.
- **R4** 비로그인 사이드바에 "체크리스트" 링크가 없다.
- **R5** 사진이 있는 공간은 썸네일을, 없는 공간은 빈 그리드를 만들지 않는다.

### 체크리스트 접근제어 (보안 핵심)
- **R6** 비로그인으로 `/checklist` 직접 접속 시 `/login?next=/checklist` 로 리다이렉트된다.
- **R7** 비로그인 상태에서 체크리스트 항목 데이터가 화면·소스에 노출되지 않는다.
- **R8** DB RLS 로 `checklist_items` 에 anon select 정책이 없어 데이터 자체가 내려가지 않는다. *(스키마 리뷰로 보증)*

### 관리자 화면 (dev-bypass 로 확인)
- **R9** dev-bypass 시 로그인 없이 `/checklist` 가 열리고 항목 목록이 렌더된다.
- **R10** 진행률(%)·평균 별점 등 요약 통계가 표시된다.
- **R11** dev-bypass 시 사이드바에 "체크리스트" 링크가 나타난다.
- **R12** "항목 추가" 로 새 체크 항목이 목록에 추가된다.
- **R13** 체크박스 토글이 진행률 통계에 즉시 반영된다.
- **R14** 별점 클릭이 반영되고, 같은 값 재클릭 시 0으로 초기화된다.
- **R15** 현재상태/요구사항 화면에서 "공간 추가" 로 새 공간 탭이 추가된다.
- **R16** seed 모드에서 편집·업로드는 저장되지 않으며 "미리보기 모드" 안내가 표시된다.

### 라이트박스 (회귀 방지 핵심)
- **R17** 사진 썸네일 클릭 시 라이트박스가 열린다.
- **R18** 라이트박스 오버레이가 뷰포트 전체(폭·높이 98%↑)를 덮는다.
- **R19** ESC 또는 닫기 버튼으로 라이트박스가 닫힌다.

### 로딩/처리 중 표시 (즉각 반응 안 되는 동작)
- **R25** 사이드바로 페이지 이동이 즉시 끝나지 않으면 해당 링크 아이콘 자리에 로딩 스피너가 표시된다(Next `useLinkStatus`). *(E2E)*
- **R26** 로그인·로그아웃·사진 업로드·공간/항목/체크리스트 추가·저장 등 비동기 동작 중에는 해당 버튼에 통일 스피너와 "…중" 문구가 표시된다.
- **R27** 첫 렌더/서버 데이터 로딩 중에는 통일 로딩 화면(`role=status`·회전 스피너)이 콘텐츠 영역에 노출된다(라우트 `loading.tsx`). seed 모드는 즉답이라 실제 지연은 Supabase 모드에서 관찰된다. *(유닛: 마크업·접근성 검증)*
- 스피너는 `prefers-reduced-motion` 시 회전을 멈추지 않되 감속한다(로딩은 필수 상태 정보).

### 핵심 로직 (유닛)
- **R20** `bundle()`: 공간별 그룹핑·sort 정렬·고아 항목 제외.
- **R21** `checklistStats()`: total/checked/pct·평균(rating>0 항목만).
- **R22** `nextSort()`: max(sort)+1 (length 기반 아님).
- **R23** `parseAllowlist`/`isEmailAllowed`: trim·소문자·fail-closed.
- **R24** 분류필터 파생: 선택 분류가 사라지면 "전체"로 폴백.

## 비범위

- 실제 Supabase/RLS 런타임 검증, 실제 비밀번호 로그인 인증 — E2E는 seed 모드로만.
- 사진 업로드의 실제 저장, 모바일 반응형, 시각적 픽셀 회귀(스크린샷 diff).
- `ProtectionLayer`(복사·우클릭 차단) — production 에서만 동작하므로 제외.
